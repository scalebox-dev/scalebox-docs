## 最佳实践

使用描述性的模板名称，例如 web-server-nginx 或 database-postgres。
避免使用通用名称如 test 或 temp。

默认情况下，模板是私有的。只有在确认安全的情况下才将模板设为公共。
公共模板可以被其他用户使用，请确保不包含敏感信息。

使用稳定的镜像标签，避免使用 latest 标签。
定期更新镜像以获取安全补丁，但请先在测试环境中验证。

为模板设置合理的默认资源限制。考虑应用程序的实际需求，
避免设置过高的资源限制导致资源浪费。
